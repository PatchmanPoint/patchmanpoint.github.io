Git-friendly by design Requests live as files, so your team can review, diff, branch, and merge.

Less weight. More flow.

Patchman focuses on the core job: create requests, organize them clearly, run them fast, and collaborate with the tools developers already use.

Use Git for collaboration

Store workspaces, collections, requests, and drafts as files. Branch, review, diff, and merge API changes like source code.

Lightweight and fast

Skip the heavy workspace feeling. Patchman is designed to open quickly and keep the request workflow responsive.

Built with Tauri and Rust

A smaller desktop app foundation with native capabilities, efficient performance, and cross-platform build potential.

Simple productivity

Move through request work with fewer pauses: open, send, inspect, commit, and keep momentum across the team.

Designed around developer files.

Patchman treats API requests as project assets, not locked workspace data. That means your API workflow can live beside your codebase, inside a repository, and inside your normal development process.

For teams, this keeps collaboration direct: create a branch, update a request, commit it, and let the team review the change.

$ git checkout -b add-login-request

$ patchman open workspace

$ git status

modified: workspaces/core/auth/login.patch

$ git add .

$ git commit -m "Add login API request"

Done. Request ready for review.
