Preparation

Preparation is a per-request JavaScript hook that runs right before the request is sent. It gets the latest draft request, lets you modify it, then sends the modified version.

How it works

  1. You write JavaScript in the Automation → Preparation tab.
  2. When you press Send, Patchman builds the request first: URL, headers, body, auth headers, and related request data.
  3. Then it runs your preparation script with request.url, request.headers, request.body, and patch.crypto.
  4. If the script is empty, the request is sent unchanged.
  5. If the script mutates request, those changes are used.
  6. If the script returns a request object, that returned object replaces the draft request.

The result must still contain:

  • url as a string
  • headers as an array
  • body as an object

What it offers

You can use Preparation for last-second request changes, including:

  • Add dynamic headers
  • Generate timestamps
  • Sign requests
  • Hash request bodies
  • Append query params
  • Rewrite URLs
  • Modify raw, form, or multipart body values
  • Prepare auth-style values that depend on the final body or URL

Crypto helpers

Preparation includes built-in crypto helpers. Both return hex strings.

patch.crypto.hmacSha256(secret, message)
patch.crypto.sha256(message)

Request shape

Headers use this shape:

request.headers.push({
  name: "X-Signature",
  value: signature
});

Body fields are available through the request body object:

request.body.mode // "raw", "formData", or "multipartFormData"
request.body.raw
request.body.formData
request.body.multipartFormData

Form fields look like this:

{
  name: "fieldName",
  value: "fieldValue",
  kind: "text" // or "file"
}

Example

const timestamp = Date.now().toString();
const signature = patch.crypto.hmacSha256("secret", request.body.raw + timestamp);

request.headers.push({
  name: "X-Timestamp",
  value: timestamp
});

request.headers.push({
  name: "X-Signature",
  value: signature
});

request.url = request.url + "?prepared=true";

return request;
