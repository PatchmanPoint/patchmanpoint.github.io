Privacy Policy

Last updated: May 26, 2026

Patchman is a local-first API client built for developers who want a simple, fast, and Git-friendly way to manage API requests.

This Privacy Policy explains how Patchman handles your data.

1. Patchman is local-first

Patchman Alpha is designed to work locally on your device.

Patchman does not connect to Patchman-owned servers. API requests are sent only to the endpoints you choose.

We do not provide user accounts, cloud sync, hosted workspaces, analytics, crash reporting, telemetry, or automatic update checks.

Your Patchman data stays on your device unless you choose to share it yourself.

2. Information we do not collect

Patchman does not collect or send us:

  • Your name
  • Your email address
  • Account information
  • Workspace data
  • Collection data
  • API request URLs
  • Request headers
  • Request bodies
  • API tokens
  • Cookies
  • Environment variables
  • API responses
  • Activity logs
  • Usage analytics
  • Crash reports
  • Device information

Because Patchman Alpha does not connect to Patchman-owned servers, we do not have access to your local Patchman data.

3. Information stored on your device

Patchman may store app data locally on your device, such as:

  • Workspaces
  • Collections
  • API requests
  • Request drafts
  • Environment variables
  • App settings
  • Local activity logs

For example, Patchman may store activity logs in an app-level local file such as:

<app data>/workspaces/.app/activity.log.jsonl

This data is stored locally so the app can work properly. Patchman does not upload this data to us.

4. API requests and responses

When you use Patchman to send API requests, those requests are sent from your device directly to the API endpoints you choose.

Patchman does not receive, proxy, store remotely, inspect, or monitor your API traffic.

Patchman does not send your API request data or response data to Patchman-owned servers.

You are responsible for the APIs you connect to and the data you send to them.

5. Git collaboration

Patchman is designed to support file-based workflows that can work well with Git.

If you choose to commit, push, export, or share your Patchman workspace files, that sharing is controlled by you.

Patchman does not control third-party Git providers such as GitHub, GitLab, Bitbucket, or self-hosted Git servers.

Be careful not to commit secrets such as:

  • API keys
  • Bearer tokens
  • Passwords
  • Cookies
  • Private environment variables
  • Production credentials

6. No analytics or tracking

Patchman Alpha does not include analytics, tracking pixels, telemetry, or usage monitoring.

We do not track how often you use the app, what features you use, or what APIs you call.

7. No crash reporting

Patchman Alpha does not automatically send crash reports, diagnostic reports, logs, or error reports to us.

Any local logs generated by Patchman remain on your device.

8. No automatic update checks

Patchman Alpha does not automatically check GitHub, our website, or any server for updates.

If you want to update Patchman, you need to do it manually through the distribution method you use.

9. No payments or licenses

Patchman Alpha does not collect payment information, license information, billing details, or customer account data.

10. Security

Patchman is designed to keep your data local, but you are responsible for protecting your own device, workspace files, and Git repositories.

We recommend that you:

  • Keep your device secure
  • Avoid committing secrets to Git
  • Use environment files carefully
  • Review workspace files before sharing them
  • Use private repositories when storing sensitive API data

11. Data retention

Because Patchman Alpha stores data locally, your data remains on your device until you delete it.

You can remove your Patchman data by deleting the relevant workspace files, local app data, or the app itself.

12. Children's privacy

Patchman is a developer tool and is not intended for children.

We do not knowingly collect personal information from children.

13. Changes to this policy

We may update this Privacy Policy if Patchman changes.

For example, this policy will need to be updated if we add:

  • User accounts
  • Cloud sync
  • Analytics
  • Crash reporting
  • Automatic update checks
  • In-app bug reports
  • Payments or licenses
  • AI features
  • Any Patchman-owned server connection

The updated version will be posted with a new "Last updated" date.
